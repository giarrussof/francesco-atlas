# PROMPTUARIO PROFESSIONALE

## PROMPT PROGETTAZIONE UdA

### P001 - UdA interdisciplinare base
```
RUOLO: Pedagogista esperto in progettazione curricolare per competenze

OBIETTIVO: Progettare UdA interdisciplinare di [N] ore per [CLASSE] su tema "[ARGOMENTO]"

CONTESTO:
- Istituto: [TIPO SCUOLA]
- Classe: [ANNO] - [N STUDENTI] totali
- BES presenti: [SPECIFICARE TIPOLOGIE E NUMERI]
- Discipline coinvolte: [ELENCARE]
- Periodo: [QUADRIMESTRE/MESE]

STRUTTURA RICHIESTA:
1. COMPETENZE TARGET (disciplinari + trasversali)
2. PREREQUISITI essenziali
3. FASI OPERATIVE dettagliate con tempi
4. METODOLOGIE ATTIVE specifiche
5. MATERIALI E STRUMENTI (incluso uso IA)
6. PERSONALIZZAZIONI per ogni tipologia BES
7. VALUTAZIONE con rubrica a 4 livelli
8. COMPITO DI REALTÀ finale

FORMATO: Schema operativo direttamente applicabile

VINCOLI:
- Utilizzo obbligatorio di almeno 2 strumenti IA generativa
- Metodologie attive al 70% del tempo
- Inclusione reale di tutti gli studenti BES
- Collegamenti con Agenda 2030
```

## PROMPT VALUTAZIONE AUTENTICA

### P004 - Rubrica competenze trasversali
```
RUOLO: Esperto in docimologia e valutazione autentica

OBIETTIVO: Creare rubrica per competenza "[COMPETENZA SPECIFICA]" con 4 livelli di padronanza

DIMENSIONI DA VALUTARE:
1. CONOSCENZE (sapere teorico)
2. ABILITÀ (saper fare pratico)  
3. ATTEGGIAMENTI (saper essere)

STRUTTURA PER LIVELLO:
- DESCRITTORE GENERALE (cosa caratterizza il livello)
- INDICATORI SPECIFICI (comportamenti osservabili)
- EVIDENZE CONCRETE (prodotti/prestazioni)
- ESEMPI PRATICI (situazioni tipo)

PERSONALIZZAZIONI:
- Adattamenti per studenti DSA
- Modifiche per studenti ADHD
- Semplificazioni per disabilità intellettiva
- Potenziamenti per eccellenze

FORMATO: Tabella dettagliata pronta per l'uso

VINCOLI:
- Linguaggio comprensibile agli studenti
- Criteri oggettivi e misurabili
- Possibilità di autovalutazione
- Feedback costruttivo integrato
```

### P005 - Compito di realtà autentico
```
RUOLO: Designer di esperienze di apprendimento autentico

OBIETTIVO: Progettare compito di realtà per [COMPETENZA/TEMA] che simuli contesto professionale reale

CARATTERISTICHE RICHIESTE:
- COMPLESSITÀ autentica ma gestibile
- APERTURA a soluzioni multiple
- SIGNIFICATIVITÀ per vita degli studenti
- SFIDA cognitiva appropriata
- COLLABORAZIONE necessaria
- PRODOTTO FINALE concreto

STRUTTURA COMPITO:
1. SCENARIO PROBLEMATICO realistico
2. RUOLI E RESPONSABILITÀ definiti
3. RISORSE DISPONIBILI (limitate)
4. VINCOLI TEMPORALI E QUALITATIVI
5. CRITERI DI SUCCESSO chiari
6. MODALITÀ PRESENTAZIONE

SUPPORTI IA CONSENTITI:
- Ricerca informazioni
- Brainstorming idee
- Verifica fattibilità
- Ottimizzazione soluzioni

VALUTAZIONE:
- Processo (come si lavora)
- Prodotto (cosa si realizza)
- Presentazione (come si comunica)
- Riflessione (cosa si è imparato)
```

## PROMPT GESTIONE CLASSE

- Regole e aspettative chiare

- Monitoraggio progressi oggettivo

## LEGENDA USO PROMPT

### Simboli e Convenzioni
- **[MAIUSCOLO]**: Campo da personalizzare obbligatoriamente
- **{opzionale}**: Informazione facoltativa
- *corsivo*: Suggerimento metodologico
- `codice`: Formato output richiesto

### Livelli di personalizzazione
- **BASICO**: Sostituire solo campi obbligatori
- **INTERMEDIO**: Adattare anche sezioni opzionali  
- **AVANZATO**: Modificare struttura per contesto specifico

### Best Practices
1. **Testare sempre** i prompt prima dell'uso in classe
2. **Iterare e migliorare** in base ai risultati ottenuti
3. **Documentare variazioni** efficaci per riuso futuro
4. **Condividere** con colleghi per miglioramento collettivo
